# Agent Working Notes

When solving Project Euler problems in this repository:

1. After completing or updating a problem, add a short entry to [learnings.md](learnings.md).
2. For each problem entry, include:
   - Goal
   - Approach
   - Learnings we actually tried/validated in code or discussion
   - Measured observations (timings, slowdown ratios, benchmark notes), when available
3. Do not include final answer/result values in [learnings.md](learnings.md).
4. Keep entries concise and focused on what was learned, not full code dumps.
5. If a previous note is wrong, correct the existing section instead of duplicating it.

This file exists to remind the agent to keep [learnings.md](learnings.md) current.
